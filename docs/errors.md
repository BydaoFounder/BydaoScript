## Обработка ошибок

BydaoScript предоставляет встроенную систему ошибок, основанную на операторе
fail и типе данных Err. Система не требует явного объявления ошибок в сигнатурах
функций — компилятор автоматически выводит Int | Err для любой функции, где
используется fail.

### Оператор fail

```
fail "текст ошибки"
fail obj, "текст ошибки"
```

fail немедленно завершает выполнение функции и возвращает объект типа Err.

Первая форма создаёт новый объект ошибки с указанным текстом.
Вторая форма добавляет текст в существующий объект obj, формируя цепочку ошибок.

Компилятор автоматически добавляет в объект ошибки имя файла и номер строки для
каждого вызова fail.

### Проверка ошибок

Любой вызов функции, которая может вернуть ошибку, требует проверки перед
использованием результата:

```
var dt = action("read")
if Err.is(dt) {
    // dt — объект ошибки
    // можно передать ошибку наверх
    fail dt, "не удалось выполнить чтение"
    // или подменить на значение по умолчанию и продолжить выполнение
    dt = DateTime.now()
}
```

### Конструкция onfail

Сокращённый синтаксис для обработки ошибки непосредственно в месте вызова:

```
var dt = action("read") onfail as e {
    // e — объект ошибки, возвращённый action
    // передать ошибку наверх
    fail e, "не удалось выполнить чтение"
}
```

Семантика:

Блок onfail выполняется, если вызов вернул ошибку, объект которой сохраняется
в переменной "e".

Если объект ошибки не нужен, можно использовать onfail в краткой форме:

```
var dt = action("read") onfail {
    log.write("что-то пошло не так")
    // вернуть значение по умолчанию
    DateTime.now()
}
```

Допускается два сценария использования:

- Прерывание функции. Если блок завершается оператором fail или return, выполнение
  текущей функции прекращается.
- Значение по умолчанию. Если последний оператор в блоке возвращает значение, оно
  возвращается как результат выполнения выражения (в примере - функции action())

### Цепочка ошибок

Каждый fail с передачей существующего объекта добавляет новый фрейм в цепочку.
Условная структура ошибки:

Err.frames = [
    { msg: "Деление на ноль",   file: "math.bds", line: 12 },
    { msg: "Ошибка вычисления", file: "calc.bds", line: 45 },
]
Для текстового представления ошибки используется функция Err.fmt(err).

### null и ошибки

null не является ошибкой с точки зрения языка. Это обычное значение, которое
разработчик может использовать для обозначения отсутствия результата. Проверка
выполняется обычным сравнением:

```
var result = find(key)
if result == null {
    // ключ не найден
}
```

### Очистка ресурсов: onexit

Поскольку fail может прервать функцию в любой точке, для освобождения ресурсов
используется блок onexit:

```
func readConfig(path: String): String {
    var file = File.open(path)
    onexit {
        file.close()
    }

    var data = file.readAll()
    // даже если readAll вызовет fail, файл будет закрыт
    return data
}
```

Блоков onexit может быть несколько. Выполняются в обратном порядке (LIFO).
