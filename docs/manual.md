# Введение

**BydaoScript** (сокращенно **BS**) - скриптовый модульный объектно-ориентированный
язык с гибкой типизацией данных и автоматическим преобразованием типов данных.

### Скриптовый

Текст на языке BydaoScript компилируется в байт-код, который затем выполняется
своей виртуальной машиной. Байт-код может быть сохранен в файле, а затем загружен
в виртуальную машину для выполнения. Часть ошибок (например, синтаксические)
обнаруживается на этапе компиляции. Часть ошибок может быть обнаружена на этапе
выполнения байт-кода (например, деление на 0).

### Модульный

Программа на BS, в общем случае, состоит из модулей. Модуль - это один файл.
Название модуля совпадает с названием файла без расширения. Модули организуются
в папки в соответствии с логикой программы:
- Пользовательские модули могут располагаться в любых папках (пути указываются
  при импорте);
- Системные модули (базовая библиотека) должны находиться в подпапке `./modules`
  относительно исполняемого файла программы.

### Объектно-ориентированный

В BydaoScript **все данные - это объекты**. Числа, строки, массивы, функции,
модули - это всё экземпляры соответствующих типов данных (классов).

>[!IMPORTANT]
>В языке нет встроенной реализации операторов (`+`, `-`, `*`, `/` и т.д.).
>Арифметические, логические и битовые операции реализованы как **методы объектов**.
>Например, запись `a + b` фактически выполняется как `a.add(b)`.

Такой подход даёт максимальную гибкость: можно определить поведение операции
"сложения" (и других) для собственных типов данных.

### Гибкая типизация

BydaoScript сочетает статическую и динамическую типизацию.

Например, объявление переменной
```
var name = 'This is string'
```
будет означать, что тип переменной `name` - строчный, т.к. переменной присвоена
строка, и компилятор будет проверять изменение типа этой переменной во всем коде программы:

```
name = 42          // ОШИБКА: тип изменился на Int, а ожидался Str
name = null        // OK: null совместим с любым типом
```

Если типы будут указаны явно, компилятор проверит их на этапе сборки:

```bs
func min(a: Int, b: Int): Int {
    return (a < b) ? a : b
}
var x = min(10, 20)     // OK, x = 10
var y = min("10", 20)   // ОШИБКА компиляции: ожидается Int
```

Функции без аннотации типов работают динамически:

```bs
func plus(a, b) {
    return a + b        // что будет — зависит от типов аргументов
}
plus(5, 10)         // 15
plus("index", 5)    // "index5"
plus([1,2], [3,4])  // [1,2,3,4] (если определён оператор + для массивов)
```

### Автоматическое преобразование типов

BydaoScript автоматически преобразует типы в выражениях, когда необходимые
преобразования типов определены в самих объектах. Например
```
var a = 5
var b = 'index' + a   // b == 'index5'
```

**Правила преобразований** определяются методами самих объектов:

- Объект может определить функцию `toType()`, которая вызывается при необходимости
приведения к типу `type`. Например, функция `obj.toStr()` возвращает строчное
представление объекта `obj`.

- Если такой функции нет — возникает ошибка (во время компиляции для статически
типизированных контекстов, иначе — во время выполнения байт-кода).

```bs
var a = getRemoteData()   // возвращает объект типа XmlNode
var b = a + 100           // ОШИБКА: XmlNode не умеет "прибавлять" число
```

## Переменные

### Объявление (var)

```ebnf
var_decl = "var" identifier [ "=" expression ]
```
Переменные объявляются ключевым словом **var**. Инициализация не обязательна.
Неинициализированная переменная принимает значение **null**.

```bs
var x               // объявление без значения
var count = 10      // с инициализацией целым числом
var name = "Bydao"  // с инициализацией строкой
```

### Присваивание

```ebnf
assign_stmt = ( identifier | member_expr ) "=" expression
```
Менять значение можно как у простых переменных, так и у членов объектов/массивов.

```bs
x = 20
person.name = "Alice"
arr[0] = 42
```

## Операторы

Особенность языка BydaoScript - обязательность фигурных скобок в операторах.
Фигурные скобки в операторах обрамляют блок кода - последовательность операторов,
даже если операторов всего один.

### Условный оператор if

Формат оператора:

```ebnf
if_stmt = "if" expression "{" block "}"
          { "elsif" expression "{" block "}" }
          [ "else" "{" block "}" ]
```
Семантика оператора аналогична условному оператору в других языках программирования.
Обязательная часть оператора - ключевое слово `if`, выражение и блок кода,
который выполняется, если выражение принимает значение `true` (истина):

```bs
if count == 0 {
    Sys.outln( 'Недопустимое количество элементов' )
}
```

Если выражение принимает значение `false` (ложь), последовательно выполняются
части оператора `elsif`, если они заданы. И если ни одно выражение не оказалось
истинным, выполняется блок кода, заданный в секции `else`.

```bs
if x > 0 {
    Sys.outln("positive")
} elsif x == 0 {
    Sys.outln("zero")
} else {
    Sys.outln("negative")
}
```

> [!IMPORTANT]
> Оператор `if` может использоваться в выражениях!

Например:

```bs
var num = Int.random(1,10)
var level = if num <= 3 { "low" }
            elsif num <= 7 { "middle" }
            else { "high" }
Sys.outln("num=" + num + ", level=" + level)
// Возможный вывод: num=5, level=middle
```

**Особенности использования в выражениях**

- Оператор **обязательно** должен иметь секцию `else`;
- Любая секция (`if`,`elsif`,`else`) может содержать произвольное количество
  операторов, но **последним** оператором должно быть выражение;
- Значения выражений во всех секциях должны быть **одного типа**.

```bs
// ОШИБКА: типы секций не совпадают (Str vs Int)
var err = if x > 0 { "positive" } else { -1 }

// OK: null совместим с любым типом
var ok = if x > 0 { 42 } else { null }
```

> [!TIP]
> В BS нет отдельного тернарного оператора (?:). Вместо него используется
> if-выражение, описанное выше.


### Оператор цикла while

Формат оператора:

```ebnf
while_stmt = "while" expression [ "next" statement { "," statement } ] "{" block "}"
```

В BydaoScript, цикл **while** имеет две формы:

-  классический цикл с предусловием. Например:

```bs
var i = 0
while i < 5 {
    print(i)   // 0, 1, 2, 3, 4
    i += 1
}
```

-  цикл с дополнительной секцией **next**, oператоры которой выполняются в **конце**
каждой итерации цикла и перед проверкой условия. Например:

```bs
var i = 0
while i < 5 next i += 1 {
    Sys.outln(i)   // 0, 1, 2, 3, 4
}
```

Внутри цикла можно использовать операторы:

- **next** — переход к следующей итерации:
  - если секция `next` определена, управление передаётся ей, затем — на проверку условия;
  - если секция `next` не определена, управление передаётся непосредственно на проверку условия;
- **break** - прерывание работы цикла и выход из блока кода


### Оператор цикла do-while

Формат оператора:

```ebnf
do_while_stmt = "do" "{" block "}" "while" expression
```

Классический цикл с пост-условием:

```bs
var i = 0
do {
    Sys.outln( 'value: ' + i )
    i += 1
} while i < 5
// Вывод
// value: 0
// value: 1
// value: 2
// value: 3
// value: 4
```

Внутри цикла можно использовать операторы:

- **break** - прерывание работы цикла и выход из блока кода, пропуская проверку условия;
- **next** — переход к проверке условия для выполнения следующей итерации.

Пример использования оператора `next`:
```bs
var i = 0
do {
    if i == 2 {
        i += 1
        next   // пропускаем вывод для i=2, сразу идём на проверку условия
    }
    Sys.outln('value: ' + i)
    i += 1
} while i < 5
// Вывод:
// value: 0
// value: 1
// value: 3
// value: 4
```

### Цикл итерации iter

Формат оператора:

```ebnf
iter_stmt = "iter" expression "as" identifier "{" block "}"
```

Оператор предлагает альтернативный вариант перебора элементов коллекции или генератора
объектов какого-то типа.

Выражение **expression** должно вернуть объект, наследник типа *Iterator*. Этот
объект-итератор присваивается переменной *identifier*, которая доступна только
внутри блока, и автоматически уничтожается по окончании работы цикла.

Для доступа к текущему элементу коллекции или сгенерированному объекту, переменная
*identifier* имеет два свойства:
- **key** - ключ (индекс), который определяет местоположение текущего элемента в коллекции;
- **value** - значение текущего элемента.

Например, перебор элементов массива:
```bs
// iter с массивом
var arr = [11, 22, 33]
iter arr as it {
    Sys.outln("arr[" + it.key + "] = " + it.value)
}
// Вывод:
// arr[0] = 11
// arr[1] = 22
// arr[2] = 33
```

Использование генератора чисел:
```bs
iter Int.range(10,15) as it {    // генератор диапазона целых чисел
    Sys.outln("Index: " + it.key + ", value: " + it.value)
}
// Вывод:
// Index: 0, value: 10
// Index: 1, value: 11
// Index: 2, value: 12
// Index: 3, value: 13
// Index: 4, value: 14
```


### Цикл перечисления enum

Формат оператора:

```ebnf
enum_stmt = "enum" expression "as" identifier "{" block "}"
```

Оператор предлагает более простой способ перебора элементов коллекции, чем
оператор **iter**, в ситуации, когда ключ (индекс) элемента не нужен - на каждой
итерации, переменная *identifier* содержит только значение текущего элемента
коллекции или значение, выданное генератором объектов. Переменная *identifier*
доступна только внутри блока и уничтожается при выходе из цикла.

Например, перебор элементов массива:
```bs
var arr = [11, 22, 33]  // статическое определение массива
enum arr as val {
    Sys.outln( 'Value: ' + val )    // выводит значение элемента массива
}
// Вывод:
// Value: 11
// Value: 22
// Value: 33
```

Использование генератора чисел:
```bs
enum Int.range(20,24) as i {    // генератор диапазона целых чисел
    Sys.outln( 'Value: ' + i )    // выводит значение числа
}
// Вывод:
// Value: 20
// Value: 21
// Value: 22
// Value: 23
```

> [!NOTE]
> - `iter` аналогичен `for...in` в JavaScript или `enumerate()` в Python — даёт доступ и к ключу, и к значению
> - `enum` аналогичен `for...of` в JavaScript или `for item in collection` в Python — только значение
>
> Используйте оператор `enum`, когда нужны только значения элементов, и оператор
> `iter`, когда требуется доступ к ключу (индексу).
