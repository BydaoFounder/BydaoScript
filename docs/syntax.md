## Введение

**BydaoScript** (сокращенно **BS**) - скриптовый модульный объект-ориентированный
язык со статической слабой типизацией данных.

### Скриптовый

Текст на языке BydaoScript компилируется в байт-код, который может быть сохранен
в файле, а затем выполняется байт-код. Часть ошибок (например, синтаксические)
обнаруживается на этапе компиляции. Часть ошибок может быть обнаружена на этапе
выполнения байт-код (например, деление на 0).

### Модульный

Программа на BS, в общем случае, состоит из группы модулей. Модуль - это один файл.
Название модуля - это название файла без расширения. Модули могут находиться в
разных папках (каталогах, директориях) в соответствие с логикой программы.
Например, системные модули, определенные в базовом комплекте языка, должны
находится в подпапке программы *"./modules"*.

### Объектно-ориентированный

В программе на BS все данные - это объекты. Числа, строки, массивы - это всё объекты
соответствующих типов данных (классов). Даже сами модули программы - этотоже объекты.
Все операции с объектами (например, арифметические или битовые) выполняются
функциями этих объектов. В компиляторе языка нет никаких встроенных операций над
объектами.

### Слабая статическая типизация

Типы данных определяются на этапе компиляции. Например, объявление переменной
```
var name = 'Bob'
```
скажет компилятору, что тип переменной *var* - символьный, т.к. переменной присвоена
строка.

А в объявлении функции, например, типы аргументов должны быть указаны явно:
```
func min( a: Int, b: Int ): Int
```
Функции могут принимать аргументы разных типов, если это указано в спецификации
функции:
```
func min( a: Int|Real, b: Int|Real ): Int
```
Зная тип значения, возращаемого функцией, компилятор может определить тип переменной:
```
var c = min(a,b)    // переменная c - целое число (тип Int)
```

Забегая вперед, можно упомянуть тип данных *Any* (любой). В этом случае,
компилятор не будет проверять соответствие типов данных, возлагая всю ответственность
на программиста. Это позволяет писать программы в стиле "динамической" типизации.

В программе на BS можно смешивать данные разных типов в одном выражении. Например
```
var a = 5
var b = 'index ' + a   // b == 'index 5'
```
Такая возможность обеспечивается функциями объектов, которые могут принимать
аргументы разных типов и преобразовывать их к требуемому типу, если аргументы
имеют функции преобразования типов.

Если объект не имеет необходимой функции приведения к типу, возникнет ошибка на
этапе компиляции.





## Программа и комментарии

Программа — это последовательность операторов и комментариев.

```ebnf
program = { statement | comment }
```

Комментарии
| Тип | Пример |
|-----|--------|
| Однострочные | `// это комментарий до конца строки` |
| Многострочные | `/* комментарий, который может занимать несколько строк */` |

## Переменные

### Объявление (var)

```ebnf
var_decl = "var" identifier [ "=" expression ]
```
Переменные объявляются ключевым словом **var**. Инициализация не обязательна.
Неинициализированная переменная принимает значение **null**.

```bs
var x           // объявление без значения
var count = 10  // с инициализацией
var name = "Bydao"
```

### Присваивание

```ebnf
assign_stmt = ( identifier | member_expr ) "=" expression
```
Менять значение можно как у простых переменных, так и у членов объектов/массивов.

```bs
x = 20
person.name = "Alice"
arr[0] = 42
```

Операторы
Блок кода — это последовательность операторов в фигурных скобках:

ebnf
block = { statement }
javascript
{
    var a = 1
    var b = 2
    result = a + b
}
Условный оператор if
ebnf
if_stmt = "if" expression "{" block "}"
          { "elseif" expression "{" block "}" }
          [ "else" "{" block "}" ]
javascript
if x > 0 {
    print("positive")
} elseif x == 0 {
    print("zero")
} else {
    print("negative")
}
Особенности:

Обязательные фигурные скобки (в отличие от C, где можно без них)

elseif — слитно (как в PHP/JavaScript), не else if

Цикл while
ebnf
while_stmt = "while" expression [ "next" statement ] "{" block "}"
Классический цикл с предусловием. Опциональная секция next выполняется в конце каждой итерации.

javascript
var i = 0
while i < 5 next i = i + 1 {
    print(i)   // 0, 1, 2, 3, 4
}
Без next вы должны сами обновлять переменную внутри блока.

Цикл iter (итерация по массиву)
ebnf
iter_stmt = "iter" expression "as" identifier "{" block "}"
Проходит по элементам выражения (ожидается массив). На каждой итерации текущий элемент связывается с переменной после as.

javascript
var numbers = [10, 20, 30]
iter numbers as value {
    print(value)   // 10, 20, 30
}
Цикл enum (итерация с индексом)
ebnf
enum_stmt = "enum" expression "as" identifier "{" block "}"
В отличие от iter, enum даёт индекс элемента, а не само значение.

javascript
var fruits = ["apple", "banana", "cherry"]
enum fruits as idx {
    print(idx)        // 0, 1, 2
    // значение: fruits[idx]
}
Важно: В текущей грамматике и iter, и enum используют одинаковый синтаксис as identifier, но семантика разная. Возможно, в будущем добавится as index, value.

Выражение-оператор
ebnf
expr_stmt = ( call_expr | "(" expression ")" )
Вызов функции или выражение в скобках могут быть самостоятельным оператором.

javascript
doSomething()     // вызов функции
(2 + 2)           // вычисление, результат игнорируется
Выражения
Приоритет операторов (от низшего к высшему)
Приоритет   Операторы   Ассоциативность
1 (низкий)  ||  левая
2   &&  левая
3   == !=   левая
4   < > <= >=   левая
5   + - левая
6   * / %   левая
7 (унарные) ! - правая
8 (высокий) () . [] — вызовы и доступ к членам  левая
Логические операторы
ebnf
logical_or  = logical_and { "||" logical_and }
logical_and = equality { "&&" equality }
javascript
if a > 0 && b > 0 || c == 0 { ... }
Сравнение и равенство
ebnf
equality   = comparison { ("==" | "!=") comparison }
comparison = addition { ("<" | ">" | "<=" | ">=") addition }
javascript
x == 5      // равно
y != 10     // не равно
age >= 18   // больше или равно
Арифметика
ebnf
addition = term { ("+" | "-") term }
term     = unary { ("*" | "/" | "%") unary }
unary    = ( "!" | "-" ) unary | primary
javascript
var result = -a + b * (c - d) / 2
var isFalse = !isTrue
Первичные выражения
ebnf
primary = identifier | number | string | array_literal
        | member_expr | call_expr | "(" expression ")"
Всё, что имеет наивысший приоритет: переменные, литералы, вызовы, доступ к членам, выражения в скобках.

Работа с данными
Массивы
ebnf
array_literal   = "[" [ array_elements ] "]"
array_elements  = expression { "," expression } [ "," ]
Поддерживается висящая запятая (trailing comma) — удобно для многострочных списков.

javascript
var empty = []
var numbers = [1, 2, 3, 4]
var mixed = [42, "hello", true]
var withTrailing = [10, 20, 30,]   // запятая в конце разрешена
Строки и интерполяция
ebnf
string        = "'" { character | interpolation } "'"
               | '"' { character | interpolation } '"'
interpolation = "#{" expression "}"
Строки можно в одинарных и двойных кавычках. Внутри — интерполяция через #{}.

javascript
var name = "Bydao"
var greeting = 'Hello, #{name}!'          // Hello, Bydao!
var expr = "2 + 2 = #{2 + 2}"             // 2 + 2 = 4
var escape = "Escaped: \#{notVar}"        // Escaped: #{notVar} (экранирование?)
Примечание: Экранирование \#{} для литерального вывода #{} в грамматике не описано, но обычно реализуется.

Числа
ebnf
number  = integer | float
integer = digit { digit }
float   = digit { digit } "." digit { digit } | "." digit { digit }
javascript
42           // целое
3.14         // дробное
.5           // валидная запись: 0.5
123.456      // обычный float
Вызовы функций / методов
ebnf
call_expr = member_expr "(" [ argument_list ] ")"
argument_list = expression { "," expression }
Функции вызываются через скобки с аргументами (разделяются запятыми).

javascript
print("hello")
math.max(10, 20)
person.getName()
arr.push(42)
noArgs()          // пустые скобки
Доступ к членам (объекты/массивы)
ebnf
member_expr = primary { "." identifier } [ "[" expression "]" ]
Доступ к свойствам через точку и к элементам через квадратные скобки.

javascript
obj.property           // точка
array[5]               // индекс
user.address.city      // цепочка точек
matrix[row][col]       // двумерный массив
data.items[0].name     // смешанный доступ
Важно: В текущей грамматике [...] опционален и только один в конце. То есть цепочки arr[0][1] формально не описаны, но могут поддерживаться через рекурсию в реализации.

Идентификаторы
ebnf
identifier = letter { letter | digit | "_" }
Правила именования:

Начинаются с буквы (латиница)

Могут содержать буквы, цифры и подчёркивания

var_name, count2, _internal — валидные имена (если _ разрешён в начале)
